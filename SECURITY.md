# Security Policy

This repository is an anonymized AWS compliance/security case study created for learning and professional showcasing.

## Reporting a Vulnerability
If you find a security issue in this repository content (e.g., accidental exposure of identifiers), please do not open a public issue.
Instead, contact the maintainer directly.

## Confidentiality Rules
This repository must never contain:
- AWS Account IDs
- ARNs
- IAM usernames
- Bucket names
- Access keys / secrets
- Internal domains / IP addresses
- Client names or organization identifiers

## Evidence Requirements
Screenshots and reports added under `0-baseline/evidence/` and `3-results/evidence/` must be anonymized:
- blur/blackout identifiers
- remove metadata where possible

## Scope
This repo contains reference notes, templates, and best-practice guidance.
It does not include production infrastructure, live environments, or client systems.

## Disclaimer
All content is provided as-is for educational purposes only.
